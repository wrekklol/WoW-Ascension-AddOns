Disable libHealComm-4.0


How do I completely disable the embedded library?
  • To disable libHealComm-4.0
  1. Navigate to the HealBot/libs directory
  2. Delete the LibHealComm-4.0 directory 
Will this option break HealBot
  • No

Why has this been made possible.
The author of libHealComm will not allow the user the choice to disable the library and so dictating that you the user will send libHealComm addon spam and parse an unfiltered combat log regardless of your view.
Due to this HealBot has been changed to safely allow deletion of the library itself, the choice is entirely yours.

When is this recommended
  • If you raid/guild leader asks you to reduce your addon spam and specifically names libHealComm
  • Your concerned this is causing you to DC, if deleting the library solves your DC issue you should report it on Curse or a WoW forum.
  • If the healers in your regular group all use HealBot and all decide to use a different Heal Comm Method. Heal Comm Method options are on the General tab, there are HealBot exclusive comms options that use up to 5 times less comms
  • Your having FPS issues and want to lighten CPU usage. In this case you should use a CPU profiler to see if removing libHealComm makes a difference on your system. HealBot has a profiler built in and can be enabled on the General tab, Titan Panel can also do this and there are other addons available specifically for this. CPU profiling is extremely CPU intensive, so after testing remember to turn it off.
    If you notice a difference in CPU usage please feedback

When is this not recommended
  • Most healers in raiding guilds will use various healing addons that depend on libHealComm
  • You are unsure of the impact on others in raid groups

What are the positives
  • Instant incoming heals are back for the caster who want to see the incoming instantly and rarely cast heals that will fail.  Use a Heal Comm Method listed with no verify.
    If "Ultra low comms - no verify" is used then incoming heal sharing will be faster to all using Ultra low comms.
  • Options to have no comms,  Use the Heal Comm Methods listed with internal
  • Options to have ultra low comms,  Use the Heal Comm Methods listed with Ultra Low comms, reduce incoming heals addon spam by up to 5 times.
  • Accurate HoT ticks. For example on HoT's that tick every 3 seconds, if you selected to show incoming heals on HoTs for the next 3 seconds, it will only show 1 tick.

What are the negatives
  • Incoming heals for channeled spells are not shown.  Direct and HoT's still work.
  • Simple heal bonuses applied from gear and talents, for example proc's are not calculated.
  • No comms with addons dependent on libHealComm


Heal Comm Method's
  1. use only libHealComm.
  2. internal no verify (No Comms) is no comms and no verify. It is for the people who asked for the return of instant incoming heals and do not want any incoming heals addon spam.
  3. internal verify cast (No Comms) again no comms but verifys the cast was successful and do not want any incoming heals addon spam.
  4. Ultra low comms - no verify. Faster/Lighter comms for healbot users. No verify instant comms.
  5. Ultra low comms - verify cast. Lighter comms for healbot users. Verify cast before comms.
  • No Verify fires the incoming heal when the cast is sent to the server, so it can start or fail.
  • Internal is no comms so only the player will see their own casts, to be completely 0 comms libHealComm must be deleted.
    or some may want to use internal no verify option for instant incoming heals and leave libHealComm embedded, this way heal information is still sent and received with others.
  • Ultra low comms is healbot exclusive and all healbot's must be set to use this method, to reduce comms libHealComm must be deleted.
For the first few minutes in a stable raid, initially Ultra low comms will generate more addon spam while it is caching new data, the amount is still very small and can now be monitored with HealBot using the Info button.
